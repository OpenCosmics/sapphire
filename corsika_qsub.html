3.4.2.5. Submit CORSIKA jobs to Stoomboot¶

In order to quickly get a good sample of simulated showers we use the Nikhef computer cluster Stoomboot to run multiple jobs simultaneously. For this purpose a modules has been written that will make this easy. The sapphire.corsika.qsub module can submit as many jobs as you want with the parameters that you desire.

The function sapphire.corsika.qsub.multiple_jobs() can do this for you. The class sapphire.corsika.qsub.CorsikaBatch sets up the environment and starts the jobs. For instance, running 100 showers with proton primaries of 10e16 eV on the standard Stoomboot queue with the default CORSIKA configuration:

multiple_jobs(100, 7, 'proton', 'stbcq')

The easiest way to run jobs is to just call the qsub.py script from the shell:

$ ./qsub.py

Changing the shower settings can be done at the end of this script after:

3.4.2.6. Corsika qsub Module¶

class sapphire.corsika.qsub.CorsikaBatch(energy=7, particle='proton', theta=22.5, queue='stbcq', corsika='corsika74000Linux_QGSII_gheisha')¶

Run many simultaneous CORSIKA simulations using Stoomboot

Stoomboot is the Nikhef computer cluster.

Parameters:
  • energy – the energy of the primary particle in log10(GeV), so an energy of 7 corresponds to 10**6 eV.
  • particle – name of primary particle, e.g. proton, gamma or iron
  • theta – zenith angle of the primary particle (in degrees), common choices: 0, 7.5, 15, 22.5, 30, 37.5 and 45.
  • queue – choose a queue to sumbit the job to: short - max 4 hours, max 1000 jobs stbcq - max 8 hours, max 1000 jobs qlong - max 48 hours, max 500 jobs, max 64 running
  • corsika – name of the compiled CORSIKA executable to use: corsika74000Linux_QGSII_gheisha corsika74000Linux_EPOS_gheisha
